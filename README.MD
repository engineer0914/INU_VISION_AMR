## 1. 개요 및 시스템 구조

### 포즈 추정 지연 원인 및 개선

* **기존 병목 현상:** 뎁스(Depth) 이미지에서 포인트 클라우드(Point Cloud)로 변환하고 연산하는 과정이 CPU 위주로 동작하여 심각한 병목 현상이 발생했습니다.
* **개선 사항:** 차후 개선된 비전 시스템에서는 무거운 포인트 클라우드 연산을 전면 배제하고, **단순 2차원 이미지 연산**으로 구조를 변경하여 처리 속도를 대폭 향상시켰습니다.

### 알고리즘 구조

* **메인 검출:** YOLOv8 Segmentation 모델
* **부가 연산:** OpenCV 기반 영상 처리 알고리즘

### 모델 학습 (Training)

유니티(Unity) 가상 공간에서 브릭 및 완성체를 다양한 조명 환경에 맞춰 배치한 후 합성 이미지를 생성하여 학습을 진행했습니다.

* **단일 브릭:** 이미지 2만 장 (YOLOv8m 모델 적용)
* **완성체(조립체):** 이미지 3만 장 (YOLOv8m 모델 적용)

## 2. 단일 브릭 검출 알고리즘

> **출력 스펙**
> * **출력 위치:** X, Y, Z [mm]
> * **출력 각도:** 12시 방향(0도) 기준
> * 정방구조(2x2) 브릭: **±45°**
> * 직사각형(4x2) 브릭: **±90°**
> 
> 
> 
> 

### 주요 검출 파이프라인

1. **객체 검출:** YOLO 모델을 통한 브릭 객체 및 마스크(Segmentation) 인식
2. **경계면 필터링 (반쯤 보이는 브릭 비활성화):** 검출된 브릭 마스크가 이미지 테두리에 걸쳐진 픽셀 수를 확인하여, 그 수치가 기준치를 초과하면 노이즈로 간주하고 비활성화합니다.
3. **뎁스(Depth) 이상치 필터링:** 검출된 브릭들의 중심점에 해당하는 뎁스 값들의 전체 평균을 냅니다. 특정 객체의 뎁스 절댓값이 평균과 크게 차이 나면 오검출로 간주해 비활성화합니다.
4. **형태 비율 검증 (2차 검증):** 브릭의 가로·세로 변의 비율을 계산하고, 해당 클래스(2x2, 4x2)의 규정된 사이즈 비율과 맞지 않으면 비활성화합니다.
5. **각도(Yaw) 산출:** 마스크를 감싸는 최소 면적 사각형(minAreaRect)을 생성하여 클래스별로 각도를 계산합니다.
* **정방구조 (2x2):** 사각형의 4개 꼭짓점 중 영상 좌표계 기준 가장 위쪽(Y값이 최소인 점)을 찾습니다. 이 점과 인접한 양옆의 두 변을 위쪽(-Y)을 향하게 둔 뒤, 12시 방향(0도)과 이루는 각도를 각각 계산하여 **절댓값이 더 작은 각도**를 반환합니다.
* **직사각형 구조 (4x2):** 사각형의 4개 변 길이를 모두 계산하여 가장 짧은 길이를 찾습니다. (픽셀 노이즈를 고려해 최단 길이의 1.05배 이하인 변을 모두 '짧은 변'으로 식별합니다.) 이 짧은 변을 위쪽(-Y)으로 향하게 둔 뒤, 12시 방향에 더 가까운 각도를 반환합니다.

## 3. 조립체(완성체) 검출 알고리즘

> **출력 스펙**
> * **출력 위치:** X, Y, Z [mm]
> * **출력 각도:** 12시 방향(0도) 기준 **±180°**
> 
> 

### 주요 검출 파이프라인

1. **객체 검출:** YOLO 모델을 통한 완성체 검출
2. **경계면 필터링:** 검출된 영역이 이미지 테두리에 닿은 비율을 확인하여 불완전한 객체는 비활성화 처리합니다.
3. **중심축 산출 (기본):** 각 클래스별로 축이 될 고유 색상을 선언합니다. (예: 배터리 - 노랑, 신호등 - 빨강 등) 해당 색상이 가장 크게 나타나는 영역을 탐색하고, 그 영역의 중심점을 산출하여 전체 중심과의 벡터 축을 생성합니다.

### 특수 처리 로직

구조상 상단을 명확히 색상으로 찾기 어려운 클래스들은 밑단을 먼저 찾고 180도를 가산(역방향 연산)하여 최종 각도를 반환합니다.

* **스몰트리 / 빅트리:** 전체 마스크에서 밑동에 해당하는 '노란색' 영역만 추출하여 객체 중심에서 아랫방향으로 향하는 축을 우선 생성합니다.
* **버거:** 객체 내 노란색(빵 부분) 영역들의 최소 면적 사각형을 생성합니다.
* YOLO 검출 마스크와 사각형 영역 내의 면적 비율(`fill ratio`) 계산
* 최외곽 윤곽선의 직선성(`straightness`) 비교 평가
* 위 항목들을 점수화하여 가장 높은 값(가장 반듯한 직선을 가진 단)을 '하단'으로 지정하고, 그 중심점을 통해 아랫방향 축을 생성합니다.

## 4. 빈 공간(Empty Space) 탐색 알고리즘

> **출력 스펙**
> * **출력 위치:** 빈 공간 중앙점의 X, Y, Z [mm]
> 
> 

### 탐색 파이프라인

1. **점유 마스크 획득:** YOLO 검출을 통해 브릭 및 조립체의 영역을 모두 합친 점유 지도(Occupancy mask)를 획득합니다.
2. **그리드 분할 및 점유율 검사:** 화면을 일정 그리드로 분할한 뒤, 각 그리드 셀(Cell) 내부에 점유 마스크가 차지하는 비율을 계산합니다. 이 비율이 임계값(Threshold) 이상일 경우 해당 셀은 안전하지 않은 공간으로 간주해 비활성화합니다.
3. **중앙 우선 탐색 (Cell Selection):** 로봇 조작에 가장 안정적인 화면 중앙 영역부터 탐색을 시작합니다.
* 영상처리 좌표계 기준으로 **`3 -> 2 -> 4 -> 1 -> 5`** 순서(중앙 ➡️ 좌우 교차)로 열(Column)을 이동하며 활성화된 빈 공간을 찾습니다.
* 해당 행의 모든 셀이 비활성화 상태라면, 다음 행(Row)으로 넘어가 동일하게 **`3 -> 2 -> 4 -> 5`** 순으로 탐색합니다. (화면 최하단인 1순위는 작업 편의상 배제)


4. **좌표 반환:** 탐색 순서에 따라 가장 먼저 발견된 활성화 영역(빈 그리드 셀)의 중앙점 픽셀에 대한 X, Y, Z 실제 위치 좌표를 반환합니다.
